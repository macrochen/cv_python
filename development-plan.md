# AI个人面试管理与模拟系统 - 开发计划 (大学生竞赛团队版)

---

### 1. 项目概述与目标

本项目旨在打造一个面向大学生求职的智能化面试辅助工具，作为“大学生智能应用开发大赛”的参赛作品。

**核心目标**：开发一个功能完整、交互流畅、能清晰展示核心AI创新点的**演示原型 (Demo)**。

**技术选型**：后端采用 **Python (Flask)** 框架，前端采用**原生微信小程序框架 (WXML, WXSS, JS) + WeUI 组件库**进行开发。

### 2. 团队角色与分工

- **指导老师**: 负责项目整体规划、系统设计、技术难题攻关、进度把控及最终演示的整体包装。
- **后端开发 (1人)**: 负责数据库设计、Flask后端API接口开发、AI模型接口集成与调用。
- **前端开发 (1人)**: 负责微信小程序所有页面的UI开发、交互逻辑实现、以及与后端API的对接。

### 5. 技术栈选择 (Technology Stack Selection)

*   **前端**: 原生微信小程序框架 (WXML, WXSS, JS) + WeUI 组件库 (原型阶段使用HTML/CSS/JS进行快速验证)
    *   **开发工具**: 微信开发者工具 (推荐使用最新稳定版)。
    *   **本地部署与预览**:
        1.  下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
        2.  在开发者工具中选择“导入项目”，指向小程序的项目根目录。
        3.  开发者工具将自动编译项目，并在内置模拟器中实时预览效果。
        4.  可通过“预览”功能生成二维码，使用微信扫码在真机上进行预览和调试。
    *   **测试与调试**:
        1.  **模拟器调试**: 开发者工具内置强大的调试器，可查看WXML结构、WXSS样式、JS逻辑、网络请求等。
        2.  **真机调试**: 通过扫码预览进入真机调试模式，可模拟真实用户环境，捕获真机特有问题。
        3.  **日志输出**: 使用 `console.log()` 在调试器中输出信息。
        4.  **用户反馈**: 使用 `wx.showToast()`、`wx.showModal()` 等API进行用户交互反馈。
        5.  **测试重点**: 由于是Demo原型，测试将主要集中在核心功能流程的打通、AI交互的模拟效果以及UI展示的流畅性。

### 6. 可行性分析与原型实现策略

#### 6.1 总体可行性

本项目作为“大学生智能应用开发大赛”的演示原型（Demo），在既定时间内实现核心功能是**完全可行**的。我们将专注于功能流程的打通和AI创新点的展示，而非生产级的健壮性或性能优化。AI功能的复杂交互将通过**模拟数据和预设逻辑**来实现，以确保演示的流畅性和稳定性。

*   **测试账户与数据**: 为了确保演示的流畅性和测试的便捷性，项目初期将建立一套**完整的测试账户和测试数据**。这些数据将覆盖所有核心功能场景，并作为前端Mock数据和后端Demo数据的基准。

#### 6.2 后端实现策略 (Python Flask)

*   **API开发**: 采用Python Flask框架，结合轻量级ORM（如SQLAlchemy）进行API接口开发。数据库将使用 **SQLite 内存数据库**，简化部署，每次应用重启数据会重置，符合Demo特性。
*   **AI模型接口集成**: 实际的LLM（大型语言模型）API调用（如Gemini API、OpenAI API）将在生产环境中实现。在原型阶段，所有涉及AI的接口将返回**预设的模拟数据**或通过简单的逻辑（如 `time.sleep()`）模拟耗时操作，以确保演示时AI响应的即时性和可控性。

#### 6.3 前端实现策略 (HTML/CSS/JS for Prototype)

*   **UI开发**: 采用HTML、CSS（Tailwind CSS）和原生JavaScript进行快速原型开发，以模拟微信小程序的用户界面和交互逻辑。这有助于快速迭代和验证产品概念。
*   **数据交互**: 前端将通过JavaScript模拟与后端API的交互，或者直接使用前端的Mock数据来驱动UI，尤其是在后端AI接口尚未完全对接时。
*   **微信登录Mock**: 为了方便测试和演示，微信登录流程将进行Mock。用户无需真实扫码登录，前端将直接模拟登录成功状态，并生成一个虚拟的用户ID和信息，以驱动后续的用户相关功能。
*   **复杂交互**: 模态框、标签页切换、动画效果等将通过JavaScript和CSS过渡来实现，力求达到流畅的用户体验。

#### 6.4 AI功能模拟策略

*   **JD关键词提取与预分析**: 前端JavaScript将包含硬编码的关键词列表和预匹配反馈逻辑，根据输入的JD（或其长度）返回模拟结果。
*   **AI定制简历生成**: 返回预设的Markdown格式简历内容，或根据用户输入关键词进行简单拼接。
*   **AI面试演练**: 预设面试问题列表，用户回答后返回硬编码的AI反馈和随机生成的评估分数。
*   **求职策略分析**: 根据选中的机会数量和类型，返回预设的Top 5关键词和公司差异分析文本。

通过上述策略，我们可以在不依赖复杂后端AI实时计算的情况下，完整展示产品的核心价值和AI赋能的求职体验。
